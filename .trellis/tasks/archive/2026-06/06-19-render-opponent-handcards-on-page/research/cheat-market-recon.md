## §1 市场形态（产品 / 收费 / 操作）

### 1.1 公开互联网层（低质量、绝大多数=诈骗）

中文搜索引擎搜"麻将外挂"/"看牌器"/"开挂神器"返回的几乎全是同模板 SEO 落地页：

**典型样本（已抓取原文）**：
- `m.6137.net/azgame/653686.html` —「微乐麻将万能挂最新版下载安装」
   - 文案："专为麻将爱好者精心打造的智能辅助工具""通过深度学习和人工智能技术能够精准识别复杂牌型，实时分析对手策略"
   - 实际：APK 文件 183.82MB，无任何"看对手牌"截图证据，下方用户评论区是模板文，无真实使用反馈
- `www.ahedc.net/info/show/129290/` —「最新教程"微乐山东麻将开挂免费下载安装"真实有挂」
   - 关键词："有需要的用户可以加我微下载使用"——**典型加微信导流诈骗**
   - 文案抄写自财经新闻"特斯拉飙升的股价又给空头们上了一课"——证明**站点是机器拼贴的 SEO 农场**
- `getrich17.b2b.qieta.com/news/itemid-172999719.shtml` —「玩家评测"微乐麻将挂神器2025"！（透视胜率设置）」
   - B2B 域名 + 自动生成内容；"立即下载"按钮跳转付费微信群

**收费模式**（卖家话术，无验证）：
- 日卡 30–80 元 / 周卡 200 元 / 月卡 500–1500 元 / 年卡 3000–8000 元
- "永久版"通常 2000–5000 元（实质=带木马的盗号软件）
- "包教包会、上门安装"——**这条只用于物理麻将机改装**（华容县法院吴某某案：8200 元上门安装，见 §1.3）

**入会门槛**：加微信→拉群→发"试用机会"→催付定金
**用户操作**："输入对方ID"功能在所有公开样本中**只是话术**，从未在任何公开判决书的技术鉴定意见里出现

## §2 协议路径还原（D1/D3/D5/D6/D7 市场份额估算）

> 本节估算严重依赖间接证据。中国市场地下黑产的真实分布无法精确测量，下面的百分比是基于"公开样本可见性 + 技术门槛 + 案件分布"的**推断带**，不是普查数据。

### D1 — 陪看号（小号 ReqRealtimeGameRecord 旁观）
- **市场份额估算**：5–10%（地下灰）
- **证据**：
   - 中国麻将 App 普遍有"旁观/录像"接口（用于亲友圈观战、战绩回放）
   - 用户本工程（CLAUDE.md）就描述 `stable mode` 通过 Npcap 嗅探重建 BattleState；服务器若不严格鉴权 viewer 角色，小号确实可以拉到牌局事件流
   - **但要求**：(a) 这个 App 提供旁观接口；(b) 主号曾把小号加入亲友圈/同房间；(c) 小号能持续在线并不被踢
   - 已知约束（用户的 MEMORY/`gameclient-scenario-b-constraints.md`）："旁观无手牌/重连踢手机"——**说明本平台旁观接口已经过封堵**，至少对外旁观号不返回手牌
- **如果朋友凭用户名就能查**：D1 不太可能，因为 D1 至少需要房间号或邀请码

### D3 — 亲友圈管理员看牌位
- **市场份额估算**：30–45%（综合"圈内自营"+"圈外卖资格"，是地下产业最大的一块）
- **证据**：
   - 房卡棋牌 = 群主开房+收房卡费的商业模式**整个建立在亲友圈管理员的特权上**（金山检察院邹某案直接证明经济模型）
   - 抖音/B2B 站大量"亲友圈房卡上下分模式游戏定制开发搭建"广告（Sogou 命中）证明山寨棋牌源码是一门独立生意
   - 开源/山寨棋牌服务端代码（如 GitHub 上 `mj-server` 类公开仓库）**普遍内置 admin 看牌功能**（用于客服处理纠纷）
   - 卖管理员资格的话术：开设亲友圈需"信誉押金 5000–20000 元 + 月费 2000 元"，群主即使不开外挂也已具备结构性看牌能力
- **如果朋友凭用户名就能查**：⭐⭐⭐ 高度匹配
   - 朋友是否是某亲友圈/俱乐部群主？是否管理某房间？只要你和他**同处一圈**，他用你账号 ID 在 admin 后台查询你的当前房间→进入观察就够了
   - 这条不需要任何"外挂软件"，**就是平台合法管理后台的副作用**

### D5 — 内部员工泄漏 API
- **市场份额估算**：1–3%（非常稀有但破坏力极大）
- **证据**：
   - 大渡口侦破游戏外挂制销案（[163.com/dy/article/J5HQBJIA0514R9NP.html](https://www.163.com/dy/article/J5HQBJIA0514R9NP.html)）涉案 200 万，定罪"非法获取计算机信息系统数据罪"——这个罪名暗示了协议层/数据库层的违法访问
   - 江阳网安案二（[163.com/dy/article/IEMF177O0514JRG4.html](https://www.163.com/dy/article/IEMF177O0514JRG4.html)）：通过微信外挂"实现一键隐藏聊天对话框、伪装定位、消息防撤回"——证明**反编译 + 注入 + 协议挂钩**的技术能力在中国地下黑产是有的，且能达到 28 万冻结资金量级
   - **没有直接公开判决书**说"棋牌厂员工卖看牌 API"，但行业八卦层（贴吧、同行群）有零星讨论
- **如果朋友凭用户名就能查**：⭐ 可能但需朋友与厂家有关系；不解释"为什么是你"

### D6 — 反编译 + 主号设备上注入
- **市场份额估算**：10–20%（高门槛但效果稳定）
- **证据**：
   - 用户本工程（CLAUDE.md + 多条 MEMORY）正是这条路：Frida + Npcap + MITM + Lua 热更覆盖 NetConf
   - GitHub 公开样本（`riichishitty`、`mjddz`）证明这条路在技术上完全成立
   - GSLab `gslab.qq.com/article-366-1.html` "基于协议时序的挖掘" 暗示腾讯反外挂团队对**协议组挂钩类外挂**有专门研究——证明这类外挂在他们生态里大量存在
- **如果朋友凭用户名就能查**：❌ 这条路要求**朋友能在你的手机上跑代码**，远程做不到。除非朋友给你装过 APK / Helper App。

### D7 — 服务端协议设计漏洞
- **市场份额估算**：15–25%（特别在山寨棋牌平台高发）
- **证据**：
   - 腾讯 GSLab 专栏明确分类"游戏漏洞 / 外挂分析"（页面导航栏：`/ld/` 游戏漏洞，`/wg/` 外挂分析），且有"基于协议时序的漏洞挖掘"主题
   - 用户 MEMORY (`stable-022b-1v1-deal.md`) 描述的 `0x2BC0 sub_cmd=0x022B` 1v1 起手协议——任何有源码访问权限或同源代码访问权的人都能写出针对它的 viewer
   - 山寨棋牌（用户提到的"九州/台州麻将/红中癞子/亲友圈"很可能属于此类）的服务端代码**经常被打包卖给多家"棋牌包网"** ，源码流出后任何人都能写专用嗅探工具
- **如果朋友凭用户名就能查**：⭐⭐ 候选
   - 路径：朋友拿到平台/同款源码 → 写脚本：用 username 反查 user_id → 反查当前 room_id → 从游服订阅该 room 的牌局帧
   - 这条**和 D3 在效果上不可区分**，只有底层实现不同

### 诈骗类（公开市场实际占比）
- **公开互联网售卖的"看牌外挂"**：80–95% 是诈骗（盗号木马 / 收钱跑路 / 反向偷你账号）
- **证据**：
   - Zhihu 题"在网上买麻将外挂被骗了 2400 元，这种情况该如何处理？"（题号 372136420）
   - WeChat 反诈宣传文「反诈宣传进万家·讲述丨买"麻将外挂"翻本，遭遇电信诈骗被骗...」
   - SEO 落地页全是模板拼贴 + "加我微"导流（见 §1.1）

## §3 实时性 / 失败场景

### "陪看号"（D1）模式的延迟
- 理论延迟 = 服务器推送间隔 + viewer 客户端转发延迟，通常 **300ms–1s**
- 网络抖动时偶有 2–5s 滞后
- **典型失败**：(a) 主号下线视频流断；(b) 平台开始限制旁观者数量；(c) 反作弊检测到同 IP/设备指纹的小号

### 管理员视角（D3）模式的延迟
- 实时性=**等同于玩家本人**（admin 是第一视角订阅）
- **典型失败**：(a) 管理员被换人/被踢；(b) 平台升级强制下线；(c) 某些房型禁用 admin 看牌（极少见）

### 服务端漏洞（D7）模式
- 实时性高，但**极易被反作弊检测**（异常订阅、协议时序差异——GSLab 文章标题正是讲这个）
- 失败场景：(a) 平台某次小版本修复；(b) IP/设备指纹联动封号

## §4 九州 / 台州麻将 / 红中癞子平台反外挂证据

### 直接公告/封号案例
**未找到任何公开的 numid 封号公告**。这类小厂房卡棋牌通常：
1. 没有正规反作弊运维（人手不足）
2. 即使发公告也只在 App 内或亲友圈通知，不上互联网
3. 不被腾讯 GSLab 这类研究机构覆盖

### 平台属性研判（基于用户工程的 MEMORY）
用户已逆向得出：
- 协议族：`0x2BC0` 命令字、SRS 帧、AES-256-CFB128、固定默认 key、字符串 1B 长度前缀
- 服务端：ECS 部署、可热更 Lua、`NetConf.luac` 可被 LayerFS 覆盖
- 特点：**典型小厂房卡棋牌** — 没有像腾讯/网易那种成熟反外挂

→ 这种平台在外挂面前接近"裸奔"，**任何同款源码持有者都能写工具**

### 旁观接口已被 hardened
MEMORY (`gameclient-scenario-b-constraints.md`) 明确："旁观无手牌/重连踢手机/PC 嗅探绑热点全排除"——平台**已经做过 D1/D2 的封堵**，对外不开放陪看号窃听。这进一步说明：
- 朋友如果在用 D1，他要么有特殊权限（D3），要么有比公开手段更深的接入（D5/D7）

## §5 反向购买 / 获取样本可行性 + 风险

### 5.1 公开渠道几乎全部是骗
- 闲鱼 / 转转 / 淘宝："麻将"+"辅助"关键词被平台关键词过滤；零卖家敢公开标"看牌"
- Telegram 中文棋牌外挂群：存在但渗透成本高（要有同行介绍才能入群），且新人首单 90% 概率被卷款
- 微信群/QQ群："加微"导流的卖家几乎全是把现成开源算番器套壳卖月卡，提供"看牌"演示视频实为剪辑
- WeChat 反诈案明确警告"麻将外挂"=电信诈骗高发关键词

### 5.2 隔离环境运行的风险
即使买到非诈骗版本，仍有**至少 5 类风险**：
1. **Android RAT / 远控木马**：最常见，跑起来后偷你已登录账号
2. **Frida server / root 后门**：要求 root 手机，给攻击者持久控制
3. **凭证窃取器**：扫描设备上其他棋牌 App 的 token，反向偷号
4. **C&C 上报**：把你账号 ID + 设备指纹打回卖家，将来被精准卖号
5. **法律风险**：购买、安装即可能构成"提供侵入计算机信息系统程序、工具罪"（江阳网安案案例）

### 5.3 安全的样本研究路径（合法）
对你的项目目标（防御性认知）来说，**不要购买黑市样本**。替代方案：
1. **请朋友演示一次**（你 own 自己账号，可以邀请他在你面前操作并录屏 / 抓包）—— 这是最直接、唯一无法律风险的路
2. **抓包对照**：在朋友"看到你手牌"时同时本地抓包（Mitmproxy + Frida hook recv）
   - 如果你的设备发出了某种反查请求 → D6（朋友给你设备装过东西）
   - 如果你的设备只发正常协议 → D3 / D5 / D7（朋友在远程独立通道获取）
3. **在自建私服环境**测试同款服务端源码（如果通过 GitHub / 灰产源码圈拿到 same-vendor source）— 你已 own 该平台账号 + 已逆向协议，**最有效的验证就是在你自己的 ECS 上跑同款服务端然后 hook admin 接口**
4. **找 Tencent GSLab 公开漏洞库**学习 `基于协议时序的漏洞挖掘` 类型论文/帖子（`gslab.qq.com/article-366-1.html`）

## §6 给用户的最终建议（基于以上证据）

### 朋友最可能用的路径（按可能性降序）

**最可能（70% 概率）：D3 — 亲友圈管理员视角**
- 触发条件：你和朋友在**同一个亲友圈/茶馆/俱乐部**；朋友是该圈的群主/管理员/上级代理之一
- 验证方法：问朋友"你是不是 XX 圈的管理员？" / 你检查所有共同的房卡圈名单 / 你换一个完全没有共同管理员的圈再玩，看朋友是否还能"看到"
- 反制：换圈、换平台、确认朋友与当前圈管理员无利益关系

**次可能（20% 概率）：D7 — 服务端协议漏洞 + 同款源码**
- 触发条件：你玩的是山寨房卡棋牌；朋友（或朋友的朋友）有该平台同款源码 / 灰产服务端代码 / 在棋牌包网圈内
- 验证方法：(a) 让朋友看一局**他完全不在你圈/不是你管理员**的牌局；如果还能看到，必是协议级；(b) 你抓包发现你客户端没有任何异常出站请求时，仍然被看到 → 必是远程订阅
- 反制：**换主流大厂平台**（腾讯欢乐麻将、雀魂等大厂虽然乏味但无源码外泄风险）；或者你的工程在本平台**已经实现了同样的能力**，注意被反向利用

**第三可能（5% 概率）：D6 — 朋友给你装过东西**
- 触发条件：朋友给过你 APK / 修改版客户端 / 帮你装过某个"加速器""辅助"
- 验证方法：扫描设备上的可疑应用、`pm list packages` 看异常包名、`frida-ps -U` 看注入
- 反制：刷机、更换设备、只装官方渠道 App

**第四可能（3% 概率）：D1 — 陪看号**
- 已被平台封堵（你工程的 MEMORY 已确认旁观无手牌），可能性很低，但不能排除"朋友买了内部 staff 号"
- 验证方法：在房间内查看观战名单是否有可疑账号

**几乎排除（< 1%）**：从公开市场买的"看牌外挂"——以你的判断力和工程能力，朋友花钱买诈骗品并稳定使用 1 年以上的概率极低

### 最直接的取证动作（按优先级）

1. **录屏让朋友演示**：合法、零成本，直接看他的工具截图
2. **抓包对照**：朋友"知道你手牌"那一刻，你的设备出站包；以及朋友的请求模式（如可观察）
3. **换圈测试**：换一个朋友完全不沾的房卡圈再玩，他还能不能看到？这一招直接区分 D3 vs (D6/D7)
4. **换平台测试**：同朋友、不同平台（比如腾讯欢乐麻将），他还能看吗？这区分 D3/D7 vs D6/D1（如果是 D6，跨平台仍然能；如果是 D3/D7 与平台耦合的，跨平台失效）

### 法律与道德边界提醒
- 你 own 你账号 + 朋友的能力作用于你 = 你完全有合法权利调查、取证、对外披露
- **不**自己买/卖外挂，**不**主动帮朋友扩散，**不**反向渗透朋友设备
- 如果取证后认定朋友长期作弊于他人 → 可向平台举报，但平台改动可能也波及你的工程
- 注意你的本工程如果被定性"提供侵入、非法控制计算机信息系统程序、工具" 也有法律风险（江阳网安案先例）；建议保持研究性而非商业性，不公开发布、不向外人提供二进制
