# 设计：ui-layout-refactor

## 当前状态
`stable_battle_panel.py` 的 `_setup_ui()` 方法中，顶部工具栏（QHBoxLayout）平铺了以下控件：
- 开始读取、停止、模拟出牌、API 设置（按钮）
- 开启 AI 分析（复选框）、AI Provider（下拉）、模型输入（文本框）
- 抓包模式（下拉：npcap/tcpdump）
- 记录本局、加入训练（复选框）
- 对手预测、动态分析（复选框）、粒子（数值）、MC（数值）、贝叶斯（复选框）、重新预测（按钮）

所有控件字体被强制设为 9px，高度 22px，非常拥挤。

右侧策略建议区域：
- `_opponent_prediction_edit`（128px 高）：对手预测信息垂直堆叠显示
- `_hard_calc_edit`（400px+ 高）：策略分析使用 HTML table 两列布局

## 方案

### 1. 顶部工具栏重构
将控件按功能域分组到 3 个 QPushButton，点击弹出 QDialog 设置面板：

**AI 按钮**：包含
- 开启 AI 分析（复选框）
- AI Provider（下拉：DeepSeek/通义千问）
- 模型名称（文本输入）

**记录+训练 按钮**：包含
- 记录本局（复选框，默认开）
- 加入训练（复选框，默认关）

**预测 按钮**：包含
- 对手预测（复选框，默认开）
- 动态分析（复选框，默认开）
- 粒子数量（数值输入，默认 5000）
- MC 次数（数值输入，默认 2000）
- 贝叶斯（复选框，默认开）
- 重新预测（按钮，直接触发）

**移除**：抓包模式下拉框（默认 npcap，从 config 读取）。

保留独立按钮：开始读取、停止、模拟出牌。

弹框实现：每个弹框使用 QDialog + QFormLayout，确认后同步状态到主面板。

### 2. 对手预测分栏展示
修改 `_format_opponent_prediction_html()`：
- 将危险牌 / 高概率持有 / 可能等待三个表格从垂直堆叠改为水平三列并排
- 使用 HTML `<table>` 实现三列布局（QTextEdit 不支持 flex/grid）
- 每列有独立标题和表格，列宽均分

### 3. 策略分析增加第三列
修改 `_format_strategy_analysis_html()`：
- 当前两列：左列（牌局状态类）、右列（分析建议类）
- 新增第三列：将"候选重排"从底部全宽区域移到右列或新列
- 实际方案：左列（状态）、中列（建议+提醒）、右列（候选重排详情）
- 使用 HTML table 三列布局，width:33% 每列
