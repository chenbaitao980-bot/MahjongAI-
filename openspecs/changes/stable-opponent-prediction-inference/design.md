# 设计：stable-opponent-prediction-inference

## 当前状态

稳定版面板已经通过 `PacketStateTracker.snapshot()` 和 `StableSimulationGame.snapshot()` 提供抓包/模拟两类状态。`game/stable_hard_analysis.py::analyze_snapshot` 当前基于我方手牌、弃牌、副露、财神、剩余牌数计算本方建议，并通过 `_opponent_predictions` 生成启发式文字。

现状限制：

- 对手预测只是文字估计，没有概率结构。
- 没有代表性隐藏手牌组合、听牌范围、等待牌概率。
- 粒子数/MC 次数不可配置。
- 模拟模式虽然有真实对手手牌，但展示预测时必须避免使用真实隐藏信息，否则与抓包模式不一致。

## 方案

新增统一推断入口：

```text
infer_opponent_hand(snapshot, config) -> OpponentPrediction
```

输入约束：

- 只能读取公开信息：我方手牌、双方弃牌、双方副露、财神、剩余牌数、对手手牌计数、当前回合、可选动作、过胡/过碰等公开事件。
- 不读取 `players[opponent]["hand"]` 的真实值。模拟模式也必须按空手牌处理。

核心算法分层：

```text
公开 snapshot
  -> 规则约束过滤
  -> 粒子采样 / Monte Carlo 后验
  -> 贝叶斯网络证据融合
  -> 结构化 OpponentPrediction
  -> StableHardAnalysis + UI 渲染
```

### 规则约束过滤

- 从完整牌墙扣除公开牌，得到剩余牌池。
- 依据对手 `hand_count` 生成候选隐藏手牌。
- 使用台州二人麻将规则检查吃/碰/杠前置合法性、财神、向听、胡牌、可选动作约束。
- 对违反已发生公开事件的粒子降权或剔除。

### 粒子 / MC 后验

- `particle_count` 控制候选隐藏手牌粒子数量。
- `monte_carlo_runs` 控制对每个候选或候选集合的后验模拟预算。
- 输出：
  - 单张牌持有概率
  - 代表手牌 Top N
  - 向听分布
  - 可能等待牌分布
  - 危险弃牌排行

### 贝叶斯网络

贝叶斯网络不替代粒子采样，而是融合证据推断“听牌范围/危险范围”：

```text
对手进度：未成型 / 二向听 / 一向听 / 听牌
花色倾向：万 / 条 / 筒 / 字牌 / 混合
牌型倾向：顺子型 / 碰碰型 / 清一色/混一色 / 财神依赖
等待类型：边张 / 嵌张 / 两面 / 字牌 / 财神相关
弃牌证据：早切 / 中切 / 晚切 / 重复切
副露证据：吃碰杠次数 / 花色集中度 / 三口风险
动作证据：能胡不胡 / 能碰不碰 / 过胡过碰锁
阶段证据：剩余牌数 / 生牌阶段 / 黄牌风险
```

贝叶斯输出与粒子后验进行加权融合。配置关闭贝叶斯时，仅显示规则约束 + 粒子/MC 结果。

## UI 设计

顶部控制条新增：

```text
对手预测 [开关]  动态分析 [开关]  粒子 [5000]  MC [2000]  贝叶斯 [开关]  [重新预测]
```

动态分析默认开启。开启后，系统先用公开信息计算证据评分，只有当评分达到阈值时才启动粒子/MC/贝叶斯预测；评分不足时展示原因并跳过计算。关闭后，只要“对手预测”开启，就按当前配置直接计算。

初版证据评分阈值为 4 分：

- 对手弃牌数：>=4 加 1，>=7 加 2
- 对手副露数：>=1 加 2，>=2 加 3
- 剩余牌数：<=50 加 1，<=35 加 2，<=24 加 3
- 当前轮到我方出牌：加 1
- 我方正在响应对手出牌：加 2
- 财神可信：加 1
- 对手手牌计数 >=8：加 1
- 对手副露花色集中明显：加 1

顶部控制行字体为上一轮压缩字体的 125%，即从 7px 调整到 9px，并同步增高控件高度。

红框位置新增“对手手牌预测”区域：

```text
可信度：中 / 样本 5000 / MC 2000 / 贝叶斯 开
进度：听牌 18%，一向听 46%，二向听 31%
高概率持有：5筒 58%｜7筒 44%｜东 39%
可能等待：4万 12%｜7万 10%
危险牌：8筒 高｜5筒 中高｜东 中
代表组合：
1. ...
2. ...
3. ...
```

长耗时推断必须后台执行，UI 主线程只显示“计算中”并在结果返回后刷新。
