# 好友房 1v1 真机帧（2026-06-16 14:13 LOLLAPALOOZA 实战抓取）

来源：ECS mahjong-tcp-proxy 日志，手机 223.104.166.229，游服代理 ECS:5767。
所有 0x2bc0 在本链路是**明文** payload（ENC= 实为明文）。内层 sub_cmd = payload[0:2] 小端。

## 发牌帧 0x022b（关键）
```
2b02 6700 10000000 00 0d 26175214125116193524295311 00 0d 53281217152833182531182319 00 0d 29263234511543351651171424 00 0d 124236284452442123315253 38 ...(尾部 0113 0134 0121 0144 0121 0125 0141 0125 0141)
```
- sub_cmd=0x022b, data_len=0x0067=103。
- 结构：`10000000` + `00 0d <13字节> 00 0d <13字节> ...`
- **instance id 唯一性检验**（同副牌每张物理牌 id 唯一，真手牌 13 个 id 不重复）：
  - block1 `26175214125116193524295311`：**无重复 → 真手牌** = `1m 2m 5m 3p 3p 3p 5s 5s 5s 6s 6s 7s 9s`
  - block2/3/4：均有重复 id → 不是手牌，是被误切的其它开局数据。
- 结论：**0x022b 只含玩家自己的 13 张手牌（首个 0d 块）**。1v1 服务器只发本人手牌。
- 解码：tile = `_GAME_INSTANCE_TILE_IDS[(byte-1)//4]`（stable/mapping.py:137），instance 顺序 条→万→筒→字。

## 其它 sub_cmd（好友房，已观测，供后续映射）
- 0x021b discard：`1b02 0600 03 42 00..` → 玩家3 打 0x42。多条：值 0x42/0x41/0x38/0x36/0x31/0x28/0x29/0x21/0x43。
- 0x021f：`1f02 0d00 03 0202 03 212121 01 21 ...`
- 0x0216 hand_update：本局**未单独出现**（金币局靠它，好友房靠 0x022b 起手 + 增量）。
- 0x0218/021a 等系统帧亦在流中。

## 与金币局差异
金币局走 0x0003(deal)/0x0216(hand_update) — 解码器已支持。好友房 1v1 改用 **0x022b 起手** + 0x021a/021b 增量，故 stable/protocol.py 缺 0x022b 分支导致 hand_raw=None。
