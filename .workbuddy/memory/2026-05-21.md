## 修复模拟出牌后弹出"过"对话框的bug

### 问题描述
用户在模拟出牌模式下，每次自己出牌后，对方出牌完毕都会弹出一个"过"的对话框，需要手动点击。这很烦人。

### 根因分析
流程：
1. 用户出牌 → `discard_self()` → 设置 pending_response 给对方
2. 对方处理 → `advance_opponent()` → 对方选择响应（碰/吃/杠/过）
3. 如果对方选择"过" → `_set_local_pending_response()` → 设置 pending_response 给我方
4. `_handle_simulation_after_opponent()` → `_prompt_simulated_response_actions()`
5. 弹出对话框，即使只有"过"一个选项

### 修复方案
在 `_prompt_simulated_response_actions()` 中增加判断：
- 如果响应动作列表中只有"过"（pass）一个选项，不弹出对话框，直接自动执行"过"
- 只有当有非"过"的可选动作（胡/碰/吃/杠）时，才弹出对话框让用户选择

### 改动文件
- `ui/main_window.py`: `_prompt_simulated_response_actions()` 方法

### 验证
- compileall 通过
- 全部 44 个测试通过

## 修复向听数计算bug（calc_shanten错误返回已胡）

### 问题描述
用户在模拟对战中，手牌为 `4m5m6m7m8m9m1p2p3p5p7p`（11张）+ 吃`5s6s7s`（1副露）+ 财神`3m`，明明没有将牌（无对子），系统却判定为胡牌（向听数=-1）。

### 根因分析
`game/shanten.py` 中 `calc_shanten()` 的情况2（无将牌候选）公式错误：
```python
shanten = need * 2 - (groups * 2 + min(taatsus + pairs, need))
```
该公式把 `taatsus`（两面/坎张搭子）和 `pairs`（对子候选）混在一起，认为它们都可以用来凑面子。但实际上：
- 当没有将牌时，`taatsus`（包括坎张搭子）**不能替代将牌**
- 只有 `pairs > 0` 时，才能用对子作将牌
- 用户手牌剩余 `5p7p` 是坎张搭子（taatsus=1），但 pairs=0，公式错误地返回 -1

### 修复方案
修改情况2分支：
1. `pairs > 0`：1个对子作将牌，剩余 `(pairs-1)` 个对子当搭子用，公式不变
2. `pairs == 0`：无对子候选，taatsus 只能凑面子，向听数不能低于1（至少需要1步凑将牌）

```python
if pairs > 0:
    shanten = need * 2 - (groups * 2 + min(taatsus + (pairs - 1), need) + 1)
else:
    shanten = need * 2 - (groups * 2 + min(taatsus, need))
    if shanten < 1:
        shanten = 1  # 无将牌时，向听数至少为1
```

### 验证结果
- 用户手牌：向听数从 -1 修复为 1（一向听）✓
- 14张胡牌：仍为 -1 ✓
- 11张+1副露胡牌：仍为 -1 ✓
- 13张听牌：仍为 0 ✓
- 12张一向听：仍为 1 ✓
- 13张+财神听牌：仍为 0 ✓
- `python -m game.shanten` smoke tests：全部通过
