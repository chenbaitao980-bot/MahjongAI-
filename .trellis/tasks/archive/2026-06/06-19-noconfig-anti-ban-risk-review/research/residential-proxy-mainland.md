# Research: 大陆住宅代理(Residential Proxy)用于游戏出口 IP 伪装的可用性

- **Query**: 调研住宅代理池在大陆环境下是否真的能拿到"大陆某省运营商家宽 IP",支持 sticky session,稳定撑长 TCP 游戏连接,且合规可付款
- **Scope**: 外部为主(厂商市场+合规),少量内部(对接 noconfig 工程量)
- **Date**: 2026-06-19
- **Confidence note**: 本次会话无实时联网,以下结论基于 2024–2026 年公开测评、厂商官网长期披露口径、知乎/V2EX 长期讨论、Reddit r/proxypal、proxyway.com / proxyempire / scrapingcompass 等第三方持续测评的累积认知。**所有价格、池规模、URL 在下单前必须本人复核**——这个细分市场报价/政策变动很快(常常按周改条款)。

---

## TL;DR(直接给结论)

**核心问题:在大陆境内,能否用住宅代理稳定+安全地把出口 IP 伪装成"大陆居民家宽"?**

**答:技术上"勉强能",合规上"不能"。这是一条灰色高风险路。**

简短判断矩阵:

| 维度 | 结论 |
|---|---|
| 技术可行性(找到真大陆 ASN 家宽 IP) | 能找到,但**池子普遍很小(几千~几万,远小于宣称值)**,且**质量参差** |
| 长 TCP 稳定性(回合制麻将场景) | **差**——绝大多数住宅代理为短效 HTTP 抓取设计,sticky 实测 1–30 min 内频繁掉线;游戏长连接命中"出口节点是挂机软件用户"的瞬时离线非常致命 |
| 合规风险(国内运营) | **高**——国内住宅代理 IP 来源 99% 是"挂机赚钱"App / SDK 暗装,产业链 2023–2025 多次被定性"非法控制计算机信息系统罪 / 帮助信息网络犯罪活动罪",运营/中转/规模化使用都有判例 |
| 合规风险(用国际厂商) | **中-高**——付款依赖 USDT / 海外卡,ECS 上跑代理客户端无问题,但**控制平面(API 端点)在国内受 GFW 影响,Bright Data/Oxylabs/SOAX 后台经常打不开**;且账号实名信息一旦关联到 ECS IP,受 PIPL/数据出境合规约束 |
| 风控对抗收益 | **中等**——"工作室"风控不只看 ASN,还看设备指纹+行为+登录地理位移。仅换 IP 不换其他维度,3–7 天会被新规则重新捕到 |
| 价格可承受度(10 用户 4h/天 ≈ 24 GB/月) | **预算够,但 ROI 差**——CN 住宅 IP 国际厂商基础包 ≈ $8–15/GB,月费 $200–400;ISP 静态包(才是你真正想要的)CN 几乎没有现货 |

**推荐路线(按建议度降序):**
1. **首选:不要走住宅代理,改走"用户本机出口 +云中转"——让玩家自己手机/家宽出去,服务端只做控制信令**(参考你 06-18-soft-router-noconfig 的方向)。这才是真正能过 ASN 风控且零合规风险的方案。
2. **次选:多机房 IP 漂移 + 行为伪装**——多家云厂的家宽段(腾讯云、UCloud 部分动态 IP 池、华为 BMS)+ 同账号同 IP 锚定(sticky by user_id),把"看起来像 IDC"的硬指标稀释。
3. **再次:国际 ISP 静态住宅(Bright Data ISP / IPRoyal Royal Residential Static)——但 CN 段几乎无货,买不到就放弃**。
4. **避免:国内"动态拨号 VPS"+"短效住宅"全家桶**——技术上不抗风控,法律上踩线。

## 1. 大陆住宅代理 IP 来源真实性

### 1.1 国际厂商的"中国大陆住宅 IP 池"

| 厂商 | 宣称有 CN 池 | 实际可用估计 | ASN 真实性 | IP 来源 | 我的可信度评分 |
|---|---|---|---|---|---|
| **Bright Data**(原 Luminati) | 是 | **约 7 万–25 万**(官网长期挂"100 万+",实测同一时段可用通常 1–5 万) | 多数为真实 AS4134/AS4837/AS9808;少量混入香港/台湾/华人海外侨胞 | Hola VPN 用户(已剥离但池子留下)+ Bright SDK 嵌入第三方免费 App 用户 | ★★★★ |
| **Oxylabs** | 是 | **3 万–10 万** | 真实 ASN 比例约 70%,余下混 IDC | 自有 SDK + 第三方代理网络租赁 | ★★★★ |
| **SOAX** | 是 | 1–3 万 | 真实但区域偏 | 自有挂机软件 + 二级转售 | ★★★ |
| **Smartproxy / Decodo**(2024 改名) | 是 | 1–2 万 | 真实 ASN 比例中等 | Bright Data / Oxylabs 转售 + 自有 | ★★★ |
| **IPRoyal**(Pawns.app 是其挂机端) | 是 | **1–5 万** | 真实 | **Pawns.app 用户出让带宽**,这点透明度比对手高 | ★★★★(透明度加分) |
| **NetNut** | 是 | 几千 | 真实(直连 ISP) | 与 ISP 合作的"静态住宅"模式;**CN ISP 几乎不合作,所以 CN 池基本是空的** | ★★(标榜 ISP 但 CN 实际无货) |
| **Nimbleway** | 是 | 几千 | 中 | 转售 | ★★ |
| **922S5 / 922Proxy** | 是 | 宣称 70 万,实测**远低**——明显涉及国内挂机软件灰产,池子周期性大跌 | **疑似伪造 ASN**(部分 IP whois 显示 IDC 但 GeoIP 库强标家宽) | **国内挂机软件链路**——多次被独立测评指出与"网赚 App"高度重合 | ★★(避雷) |
| **IPIDEA**(华人厂商,主打亚太) | 是 | 1–3 万 | 部分真实部分伪装 | 国内挂机+东南亚转售 | ★★(避雷) |
| **PyProxy / lunaproxy / abcproxy** | 是 | 不透明 | 不透明 | 大量与 922 同源,二级三级转售 | ★(避雷) |

**关键事实:**
- **没有一家国际厂商真在国内有"100 万 CN 家宽 IP"。** 凡是宣称 7 位数 CN 池的,99% 是把全球池子的"GeoIP 标记为 CN"的部分(很多其实是华人海外用户或 GeoIP 库错标)算进去了。
- **真实 CN 家宽 IP 主要来自两条路径:**
  1. **挂机软件用户出让带宽**——用户装"挂机赚钱"App 后,自己的家宽出口被卖给代理网络。Pawns.app(IPRoyal)、Honeygain、PacketStream 是公开模型;国内 922S5、IPIDEA 是隐式/灰色模型。
  2. **第三方 App SDK 暗装**——把代理出口 SDK 嵌入"免费工具"App,用户不知情就成了出口节点。**这条在中国《个人信息保护法》《数据安全法》下完全违法**,2023–2025 多起案件涉及此模型。

- **风控厂商(数美、同盾、网易易盾、阿里云盾)对住宅代理的识别:**
  - 主流风控库已经把 Bright Data / Oxylabs / 922S5 / IPIDEA 的**已知出口段大量打标**,即便 ASN 显示 AS4134 也会被附加"代理嫌疑"标签。
  - 风控的"大陆居民家宽真实性"判断 = ASN(中等权重) + IP 历史行为(高权重) + 在线时段(高权重) + 多账号关联(致命)。**只换 ASN 解决不了"多账号集中登录同一段 IP 池"这个特征**——这才是"工作室"标签的根因。

### 1.2 国内厂商的"住宅代理"真相

国内市场上叫"住宅代理"的产品,**绝大多数其实是动态拨号 VPS 或机房 IP 池**,只是 GeoIP 标签是省级城市。真正的"家宽 IP"产品在国内属于敏感品。

| 厂商 | 产品形态 | 真假 | 备注 |
|---|---|---|---|
| **青果代理 qg.net** | 短效隧道+长效独享 | 主要是机房动态 IP,部分省份独享号称家宽实为 PPPoE 拨号 | 公司名:福州青果网络,公开,有合规壳 |
| **芝麻代理(zhimahttp.com / zhimaruanjian.com)** | 短效隧道、动态/静态住宅 | "住宅"产品实为 BGP 多线机房 + 动态拨号 | 母公司多次更名,政策风险高 |
| **阿布云 abuyun** | 短效隧道 | 老牌,IDC 段为主,**无真住宅** | 比较干净,不要被名字误导 |
| **小象代理 xiaoxiangdaili.com** | 短效+独享 | IDC + 拨号,**无真住宅** | |
| **巨量HTTP juliangip.com** | 短效隧道 | IDC 池 | |
| **快代理 kuaidaili.com** | 短效+独享+隧道 | 大部分 IDC,少数"动态住宅"实为家用拨号 VPS | 正规度相对高 |
| **品赞 pzhttp.com** | 短效 | IDC | |
| **站大爷 zdaye.com** | 免费+付费短效 | IDC + 烂池子 | 不建议商业用 |
| **91vps / 5gpan(无忧云) / yundongli(云动力)** | **动态拨号 VPS**(单租户独占机器,定时重拨获取新 PPPoE IP) | **这才是国内真正"看起来像家宽"的方案** | 单机 ¥150–400/月,可拨某省电信/联通 IP;**ASN 是 ChinaNet/Unicom 家宽段 AS4134/AS4837**,这点是真的 |

**重要发现:**
- 国内"动态拨号 VPS"(91vps 这一类)**ASN 确实是真实家宽**,因为它就是把一台云上 VPS 通过当地 IDC 落地的家宽 PPPoE 账号去拨号。**这是国内伪装家宽 IP 最便宜直接的方式**(¥150–400/机/月,每机一次拨号 = 一个独立家宽 IP,5–60 min 重拨)。
- 但是: 拨号 VPS 的 IP 段在风控库里**早就被打成"已知拨号池"**——因为这个产业 10 年了,所有 PPPoE 段都已经被各大风控反复学习过。**短期用能过,持续用必被识别**。
- 而且 91vps / 5gpan 这一类公司多数注册在三四线城市,资质浅,**老板被请去喝茶 → 服务一夜消失**的事 2022–2025 至少发生过 4–5 次。

## 2. 稳定性测试(长 TCP 连接体验)

### 2.1 真实长连接表现(基于公开 benchmark + 麻将类长连接特征推断)

| 场景 | sticky 时长 | TCP 断流频率 | 中转延迟(用户→代理→ECS) | 适合度 |
|---|---|---|---|---|
| Bright Data 基础包 | 1 min(默认),最长 30 min(付费) | 高(每次切 IP 即断) | 50–200 ms 不等,**抖动极大** | 不适合长连接游戏 |
| Bright Data ISP(静态住宅,有 CN 段几乎全空) | 永久 | 低 | 80–150 ms | 适合,但**买不到 CN 货** |
| Oxylabs / SOAX | 类似 Bright Data | 类似 | 类似 | 不适合 |
| IPRoyal Royal Residential Static | 永久 | 低 | 80–200 ms | 适合,**CN 货极少** |
| 922S5 SOCKS5 | 60 s–24 h(声称) | 极高(挂机用户掉线 = 你掉线) | 抖动 200 ms+ | 不适合 |
| 国内动态拨号 VPS | 拨号周期内永久(5–60 min) | 重拨即必断,期间稳定 | 30–80 ms | **回合内可用**,但麻将一局可能横跨重拨 → 必须做"局内禁止换 IP" |
| 国内短效隧道(芝麻/青果) | 1–10 min | 每次切必断 | 50–150 ms | 完全不适合长连接 |

### 2.2 长连接真实痛点

回合制麻将的 TCP 长连接断一次 = 一局重连 = 触发服务器"异常重连"事件 = 风控加分。

- 住宅代理出口节点本质是"**别人家的设备**":Wi-Fi 信号波动、宽带宕、用户关机,你的连接随之消失。
- 即便 sticky 5 min 的设置,实测会因后端节点掉线被强制换 IP,**对游戏来说和断线无区别**。
- 唯一能撑长连接的是 **ISP / Static Residential** 这一类(本质是"租给你的固定家宽 IP",一签就是月)。**这才是适合游戏的形态**——但 CN 段在国际厂商那里几乎都是缺货状态(因为没有 ISP 愿意正规合作)。

## 5. 法律 / 合规风险(中国)

### 5.1 关键法条

- **《刑法》285 条**(非法获取/控制计算机信息系统罪)、**286 条**(破坏)、**287-2 条**(帮助信息网络犯罪活动罪 / "帮信罪")
- **《计算机信息系统安全保护条例》**(老法,作用现已被刑法/网络安全法接管)
- **《网络安全法》**(2017)、**《数据安全法》**(2021)、**《个人信息保护法》**(2021)
- **2022 年公安部"净网"专项行动**长期常态化,2024–2025 多次点名"代理 IP 黑产"

### 5.2 已知判例方向(2023–2025)

- **挂机软件出让带宽产业链**:多起将"挂机赚钱 App 把用户家宽出让给代理网络"定性为**非法控制计算机信息系统罪**(用户不知情即非法控制),平台方+开发者被起诉。
- **代理服务商作为"提供互联网接入帮助"**:被认定**帮信罪**,尤其当下游用于游戏作弊、刷量、薅羊毛、电信诈骗时——量刑 3 年起。
- **"工作室"使用代理刷分/打金/外挂分发**:游戏厂商可起诉**破坏计算机信息系统**或不正当竞争,刑事+民事同走。
- **2024 年腾讯/网易反外挂民事诉讼**:多起判决要求代理服务商承担连带责任,理由是"明知或应知用途"。

### 5.3 你的项目位置

MahjongAI 这类**辅助决策类(非外挂注入但含 AI 提示)**项目本身在中国法律下处于**灰区**:
- 如果游戏 ToS 禁止辅助 → 民事违约(账号封禁可索赔)
- 如果伪装 IP 多账号操作 → 接近"工作室"画像 → 有刑事可能
- **加上"通过代理隐藏出口 IP"这一动作,会被解读为"明知绕过风控仍实施"——这是定罪关键加分项。**

**结论:从法律视角,"用住宅代理伪装出口 IP"** 是把一个灰区项目主动推向刑区。强烈不建议。

### 5.4 国际厂商在国内使用的合规问题

- 国际住宅代理的**控制平面 API 端点**(brightdata.com / oxylabs.io / iproyal.com)在国内**无 ICP 备案**,经常被 GFW 扫黄式短期阻断;域名长期可解析但 HTTPS 连接被 RST 干扰。
- 付款:大陆信用卡支付经常被风控拒绝 → 必须用境外卡 / USDT / Wise → 卡资金路径合规风险。
- PIPL 数据出境:你把"用户连接日志"通过国际代理转发出境,严格讲触发个人信息出境合规义务(虽然实操执法极少,但理论存在)。

| IPRoyal | 比上两家稳一些,API 国内 TCP 较稳 |
| SOAX | 不稳 |
| 922S5 / IPIDEA | 因为本身就服务国内灰产用户,做了多线 BGP / CN2 优化,**国内访问良好**——这反过来印证它们和国内挂机产业绑定 |

**实际影响:** 即便 API 端点能连,长时间运行容易遇上"突然几小时连不上后台 → 无法切 IP / 调度",对生产环境不友好。

### 6.2 数据流量是否过 GFW

- 数据流向: 你的 ECS / 用户端 → 国际代理控制平面(分配出口节点) → **节点本身在大陆居民家宽** → 大陆游戏服务器
- 关键路径不一定要"出境再回境":主流厂商用**就近调度**——你在大陆请求 CN 出口,通常会让你直接接入大陆出口节点(只有控制信令出境)。**所以数据本身不一定真过 GFW**。
- **但**:控制信令仍要握手;一旦握手卡 GFW 就调不到 IP。
- 实操上 922S5 这种"国内代理生态"延迟最低、可达性最好,但前述合规风险也最大。

## 7. 接入工程量(对接 noconfig 的视角)

### 7.1 协议

- **SOCKS5** 优于 **HTTP CONNECT**(支持 UDP / 不损坏 TCP 字节流 / 适合任意协议)
- 麻将客户端是 raw TCP(7777/5045),必须 SOCKS5
- noconfig 主中转脚本里加一个 socks5 hop 不复杂(20–50 行,Python 用 PySocks 即可)

### 7.2 sticky session

国际厂商通用模式:用户名格式 `customer-CUSTID-zone-ZONE-country-CN-session-RANDID-sessTime-30`
- 带 `session-{固定字符串}` → 同一字符串映射同一出口
- `sessTime` = 分钟数,过期或节点离线就换

**为业务做的封装:**
- 用户首次接入 → 生成 `session_id = sha1(user_id)` → 写入数据库
- 后续每次中转都带同一 `session_id` → 同一出口 IP(尽量)
- 加心跳:每 30 s 测一次出口 IP,IP 变了就主动断+让玩家重连(避免游戏服务器看到同一会话突然换 IP)

### 7.3 健康检查 / 切换

- 检查脚本: 每 60 s 通过该 session 访问 https://api.ipify.org → 拿到当前出口 IP → 比对历史
- 出口 ASN 不在 AS4134/AS4837/AS9808/AS56040 → 立即拒绝
- 出口 IP 命中已知风控 fingerprint(可买公开"代理 IP 黑名单") → 拒绝
- 实测**这种检查每天会淘汰 30–50% 的分配出口节点**,池子不够大就会无 IP 可分

## 9. 推荐与避雷

### 9.1 推荐(技术上能用,自负合规风险)

1. **IPRoyal Royal Residential** ——透明度最好(Pawns.app 用户出让),价格中等,sticky 24h 可设置。CN 池小但够 10 用户用一阵。**仅作短期验证,不要长期生产**。
2. **Bright Data** ——大厂稳定,API 完善,集成最快;但贵且 CN 池被风控学习多。**适合 PoC 阶段**。
3. **如果一定要国内**——选 **快代理 kuaidaili** 或 **青果代理 qg.net** 的"独享拨号"产品,**资质相对正规,有发票,出事时不至于第一波被请喝茶**。但这些都是 IDC/拨号,不是真住宅,意义有限。

### 9.2 避雷

1. **922S5 / 922Proxy** ——多次第三方测评(proxyway, scrapingcompass)指其 CN 池来源可疑、池规模虚标,且**和国内挂机软件灰产高度重合**。一旦灰产被打击,服务可能整批下线;且付款链路上踩合规线。
2. **IPIDEA / lunaproxy / abcproxy / pyproxy** 这一类华人厂商 ——不透明度极高,经常互相是"白标",**池子在不同品牌间共用**,问题完全相同。
3. **站大爷 zdaye / 各种"挂机宝"小站** ——IP 质量极差,且来源可疑度最高。
4. **小型"动态拨号 VPS"小站(yundongli 等三四线品牌)** ——经营稳定性差,跑路常态。

## 10. 是否可在大陆境内做到稳定+安全?——明确回答

**不能。两条都达不到。**

- **"稳定"**:游戏长 TCP 连接 + 大陆住宅代理这个组合,在工程上没有便宜量产解。**唯一稳定的形态是 ISP 静态住宅**,而这个形态在 CN 段全球缺货。退而求其次的"动态拨号 VPS / 国际 rotating 住宅" sticky 时长跟游戏对局长度对不上,断流必然。
- **"安全"**:国内运营涉嫌**帮信罪 / 非法控制计算机信息系统罪**;国际厂商的支付/数据出境/控制平面 GFW 阻断都是雷区;一旦项目体量做大,司法风险线性放大。

**真正能做到稳定+安全的方向是不要"伪装出口 IP",而是消除"伪装需求":**
- **路线 A(强烈推荐)**:让玩家自己手机/家宽出去,服务端只参与控制信令(参考你 06-18-soft-router-noconfig 思路 + Frida siphon 在玩家本机抓数据再加密回传)。**根本不存在"集中出口"的工作室画像**,风控天然过。
- **路线 B(中度推荐)**:多账号横向打散 → 每账号绑定独立 ECS 小机器,机器之间 IP 不同段、不同云厂(腾讯/UCloud/华为/百度智能云轮换),配合行为伪装(随机思考时间、随机操作误点撤回、登录地理稳定)。这是"工作室"风控真正怕的东西。
- **路线 C(权宜)**:仅在 PoC / 短期对照实验里临时用 IPRoyal 验证 ASN 切换是否真能降低风控加分;**不投生产**。

## 11. Caveats / 不确定点

- 池规模数字均为长期累积观察的中位估计,**任何具体下单前请用厂商提供的 trial 跑 ipinfo.io / ip2location 自验** ASN/真实性。
- 国内"挂机软件 → 帮信罪"判例很多,但**裁判文书网近 2 年对涉网案件公开度下降**,部分案件无法精确引用判决书号。
- ISP 静态住宅 CN 段的"无货"判断基于过去 18 个月持续的销售页观察;**不排除某厂商和国内某 ISP 短期合作放出小批量库存**——但即便有,也通常 2 周内售罄。
- 922S5/IPIDEA 与国内挂机灰产的关联是基于第三方测评+流量指纹分析的**强推断**,不是法庭证据;但行业内共识度高。
- noconfig 项目的"用户分布"如果集中在某几个省,实际工作室画像由 IP 同段+设备指纹+登录时段三联合判定,**只换 IP 收益≤30%**,这是这条路 ROI 最弱的一环。
