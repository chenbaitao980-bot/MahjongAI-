## ui-layout-refactor Bug 修复（qt-dialog-widget-reparent）
- 问题：弹框内直接使用主面板控件（self._opponent_prediction_checkbox 等），弹框关闭后 Qt 销毁这些控件，后续访问崩溃
- 报错：`RuntimeError: wrapped C/C++ object of type QCheckBox has been deleted`
- 根因：Qt QDialog 关闭时销毁布局内所有子控件。通过 layout.addRow() 添加已有控件会改变其父对象，弹框关闭后控件被销毁
- 修复：弹框内创建临时控件，从主面板读取初始状态，用户点击 OK 后同步回主面板控件
- 教训：Qt 弹框中永远不要直接使用外部已存在的 QWidget 实例，必须创建临时控件做数据绑定
