# 设计：stable-only-capture-module

## 当前状态
`MainWindow._setup_ui()` 直接把多个旧模块和稳定版都挂到同一个 `QTabWidget`。稳定版抓包线程 `StableCaptureThread`、事件处理、Excel 记录、分析 worker 等逻辑目前也集中在 `ui/main_window.py` 内。旧视觉模块虽然可以继续存在，但入口展示会干扰用户只使用稳定版抓包模块。

## 方案
1. 主界面改为稳定版单入口：默认只构建并显示 `_build_stable_tab()`，窗口标题和状态栏保持现有应用语义。
2. 旧模块不在主界面展示：不创建旧 tab，不主动初始化旧 UI 面板，避免用户进入非稳定版流程。
3. 稳定版代码隔离：优先将稳定版抓包线程和稳定版事件协调逻辑从 `MainWindow` 抽到稳定版专用模块；如果抽离过程中发现依赖过重，则先做最小隔离层，保证稳定版路径独立且可运行。
4. 保留旧代码文件：不删除 `ui/battle_panel.py`、`ui/capture_panel.py`、`ui/collection_panels.py` 等旧模块，避免影响导入、测试或后续回滚。

## 业务规则处理
- 原 Requirement: `stable-reader` 的抓包采集、协议解码、分析门控、兼容性。
- 本次处理方式: MODIFIED，同一稳定版能力下收束 UI 入口和代码边界。
- 不改变抓包协议、解析规则、分析门控或训练数据规则。

## 回滚方案
恢复 `MainWindow._setup_ui()` 中旧 tab 的创建，并把抽出的稳定版协调逻辑重新接回或保留为兼容模块。

## 追加设计：窗口行为优化

当前 `MainWindow.__init__()` 调用了 `self.setWindowFlag(Qt.WindowType.WindowStaysOnTopHint)`，这会让窗口在用户切换到其他软件时仍保持置顶。应移除该 flag，让主窗口使用 Qt 默认普通窗口行为。

缩放优化方案：

1. 主窗口不设置置顶 flag。
2. 主窗口设置合理的 `setMinimumSize(width, height)`，保留可变大小，不使用固定宽高。
3. 稳定版面板继续使用布局伸缩比例，不把根容器固定宽度。
4. 固定宽度只保留在输入框、下拉框等工具栏控件上，避免它们把根窗口锁成不可横向缩放。

回滚方式：恢复 `WindowStaysOnTopHint` 或移除新增的最小尺寸设置。

## 追加设计：模拟出牌

新增模拟模式作为稳定版的“非抓包数据源”。它不修改协议解析和 `PacketStateTracker`，而是通过独立模拟器产出与 `PacketStateTracker.snapshot()` 同形状的数据，让现有稳定版 UI、硬算分析、对方预测、推荐展示继续复用。

### 模拟状态模块

新增 `stable/simulator.py`：

1. 初始化 2 人局模拟牌墙：
   - 使用标准 34 种牌，每种 4 张。
   - 随机选择财神。
   - 我方 13 张、对方隐藏手牌计数 13 张。
   - 自动给我方摸 1 张，使开局轮到我方出牌，并满足 14 张分析门控。
2. 维护状态字段：
   - `local_player=0`、`opponent_player=1`、`current_turn`、`remaining_tiles`、`baida_tile`、`hand_trusted=True`、`baida_trusted=True`、`turn_trusted=True`。
   - 我方 hand/discards/melds；对方 hand 隐藏为空、hand_count 维护计数、discards/melds。
   - `events` 记录模拟事件流。
3. 输出 `snapshot()`，形状与 `PacketStateTracker.snapshot()` 兼容。
4. 提供 `to_battle_state()` 或等价方法，使 MainWindow 可复用现有 `BattleAnalysisThread`。
5. 提供 `discard(tile_id)` 和 `advance_opponent()`：
   - 我方出牌后进入电脑回合。
   - 电脑回合采用简单规则：摸一张，按本地硬算/随机兜底选择一张弃牌，然后轮到我方摸牌。
   - 轮到我方时保证可展示 14 张手牌和推荐。

### 出牌弹框

新增独立出牌弹框：

1. 只在模拟模式且 `current_turn == "self"` 时弹出。
2. 读取当前我方手牌，展示可选牌按钮或列表。
3. 可预选当前硬算推荐牌，但用户仍可手动选择。
4. 用户确认后调用模拟器 `discard(tile_id)`。

### UI 接入

`StableBattlePanel` 顶部增加“模拟出牌”按钮和信号。`MainWindow` 增加模拟模式字段：

- `_stable_simulator`
- `_stable_simulation_active`

点击模拟出牌时：

1. 停止真实抓包线程。
2. 初始化模拟器。
3. 刷新 `set_snapshot()`。
4. 启动同一套稳定版分析流程。
5. 弹出出牌框。

停止按钮：

- 如果处于模拟模式，结束模拟并清理状态。
- 否则保持原真实抓包停止逻辑。

### 边界

- 不引入完整麻将吃碰杠胡规则，先实现可轮转、可出牌、可分析的 2 人模拟对战。
- 不展示对方隐藏手牌明文，保持 stable-reader 隐藏手牌约束。
- 不影响真实抓包路径。

## 追加设计：模拟界面舒适度

1. 主窗口尺寸：
   - 默认 `resize` 调大到更适合稳定版左右双栏的尺寸。
   - 最小尺寸同步上调，但仍保持可缩放，不使用固定尺寸。
2. 策略建议区：
   - 将上方摘要文本框高度压缩为紧凑摘要。
   - 将硬算/策略文本框作为主要空间承载区，减少红框区域中的无效留白。
3. 出牌弹框：
   - 移除下拉框。
   - 使用按钮网格平铺当前全部手牌，类似全键盘。
   - 推荐牌按钮使用醒目样式或文字提示。
   - 点击按钮即选择并关闭弹框；取消按钮只用于暂停本次出牌。

## 追加设计：模拟出牌确认

修正上一版交互：全手牌按钮网格不再“点击即出牌”，而是采用选择 + 确认。

1. `SimulatedDiscardDialog` 保存 `_selected_tile` 和当前选中按钮。
2. 手牌按钮点击后：
   - 设置 `_selected_tile`。
   - 更新按钮样式为选中状态。
   - 启用底部“出牌”按钮。
   - 不调用 `accept()`。
3. 底部按钮：
   - “出牌”：仅当 `_selected_tile` 非空时 `accept()`。
   - “暂停”：`reject()`，不改变模拟局。
4. 关闭窗口等同“暂停”，MainWindow 不推进模拟局。

回滚方式：恢复按钮点击直接 `accept()`。

## 追加设计：模拟吃碰杠胡事件

当前 `StableSimulationGame` 只做摸打轮转，`melds` 永远为空，且没有胡牌结算。追加事件能力以“基础可用”为目标，不做完整台州麻将番型、包牌、抢杠等复杂结算。

### 合法动作判断

在 `stable/simulator.py` 中新增动作判断：

1. 胡：
   - 自摸：当前玩家摸牌后，使用 `hand_to_counts()` + `calc_shanten()`，若结果为 `-1` 则可胡。
   - 点炮：某玩家打出的牌加入另一方手牌后，若 `calc_shanten() == -1`，另一方可胡。
2. 碰：
   - 对方打出某牌后，当前玩家手中同牌数量 >= 2。
3. 明杠：
   - 对方打出某牌后，当前玩家手中同牌数量 >= 3。
4. 暗杠：
   - 当前玩家回合手中同牌数量 >= 4。
5. 吃：
   - 仅普通数牌可吃；对方打出某数牌后，当前玩家手中存在能与该牌组成顺子的两张牌。

### 动作应用

1. 胡：
   - 设置 `phase="finished"`、`current_turn="none"`。
   - 事件流记录“我方胡牌/对面胡牌”。
2. 碰：
   - 从玩家手牌移除两张同牌。
   - 从对方最后弃牌中移除被碰牌。
   - 写入 `melds: {"type": "pon", "tiles": [tile, tile, tile]}`。
   - 行动玩家进入出牌回合。
3. 明杠：
   - 从玩家手牌移除三张同牌。
   - 从对方最后弃牌中移除被杠牌。
   - 写入 `melds: {"type": "kan_open", "tiles": [tile]*4}`。
   - 行动玩家补摸一张后进入出牌回合。
4. 暗杠：
   - 从玩家手牌移除四张同牌。
   - 写入 `melds: {"type": "kan_closed", "tiles": [tile]*4}`。
   - 行动玩家补摸一张后继续出牌回合。
5. 吃：
   - 从玩家手牌移除组成顺子的另外两张牌。
   - 从对方最后弃牌中移除被吃牌。
   - 写入 `melds: {"type": "chi", "tiles": sorted_sequence}`。
   - 行动玩家进入出牌回合。

### UI 交互

1. 我方可选动作：
   - 新增 `SimulatedActionDialog`，显示胡/吃/碰/杠/过。
   - 我方出牌前如果可暗杠/自摸胡，先提示动作。
   - 对方出牌后如果我方可胡/吃/碰/杠，提示动作；选择“过”则继续我方摸牌。
2. 电脑动作：
   - 电脑自动选择优先级：胡 > 杠 > 碰 > 吃 > 过。
   - 电脑副露写入对方 `melds`，仍不展示对方隐藏手牌明文。

### 边界

- 不做完整计分。
- 不做复杂抢杠胡、包牌、不死包等结算。
- 吃碰杠胡事件只用于模拟训练/交互体验，不影响真实抓包解析。
