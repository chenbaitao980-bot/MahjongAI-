# stable-capture-training-data

## 为什么

长期目标是让 MahjongAI 在真实使用中持续积累经验：先把稳定抓包得到的局面、动作、结果沉淀为可回放、可标注、可训练的数据，再用这些数据做监督学习，最后接入自博弈强化学习。

当前项目已经有 `mc_simulator`、`mc_policy`、`shanten`、`ukeire`，也已经推进了稳定抓包读取和硬算面板。下一阶段不应直接跳到 Mortal/Suphx 式自博弈，而应先保证抓包数据稳定、同源、可回放、可纠错，否则训练会把解析噪声学进去。

## 影响面

- `stable-reader` 能力：把抓包稳定版输出定义为训练数据源，而不仅是 UI 展示源。
- `excel-logger` 能力：继续作为人工复盘和纠错入口，但不把 xlsx 当作唯一训练格式。
- 后续代码范围预计包括 `stable/`、`scripts/replay_stable_reader.py`、`game/excel_logger.py`、`data/` 下的数据导出脚本，以及测试。

## 改动范围

本 change 先做规划，不修改运行代码：

1. 定义稳定抓包训练数据闭环。
2. 定义监督学习数据最小字段。
3. 定义自博弈强化前置条件。
4. 明确第一阶段验收标准。

## 验收

- [ ] 规划明确区分抓包稳定版、监督学习数据、自博弈强化三个阶段。
- [ ] 规划要求训练样本必须来自可信 `snapshot()`/events 回放，不能来自 AI 文案。
- [ ] 规划要求数据不足、未知映射、财神未可信、回合未可信时不得产出训练动作标签。
- [ ] `gitnexus detect-changes` 无异常范围外变更。
