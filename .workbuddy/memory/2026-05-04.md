## 副露区识别问题修复

**问题**：台州麻将吃牌后的副露（如二万三万四万）显示在手牌左侧，被系统误识别为手牌而非副露区。

**根本原因**：
1. `vision/pipeline.py` 中 `meld_count` 永远为 0（`self_player.melds` 从不被自动填充）
2. `vision/layout.py` 的 `hand_region()` 只支持副露在右侧时收缩右边界，不支持副露在左侧时向右偏移左边界

**修改内容**：

### `vision/layout.py`
- `hand_region()` 新增 `meld_side` 参数支持（读取自 `settings.yaml`）
- `meld_side == "left"`（台州麻将）：x 向右偏移 `meld_count * meld_unit_w`，w 等量缩减
- `meld_side == "right"`（默认）：仅缩减 w（原逻辑）

### `vision/pipeline.py`
- `_process_frame()` 中 `meld_count` 计算增加优先级：
  1. 先用 `state.self_player.melds`
  2. 再继承 `prev_state.self_player.melds`
  3. 最后调用新增的 `_detect_self_meld_count_visual()` 视觉估算
- 新增 `_detect_self_meld_count_visual(frame)` 方法：
  - 只在 `meld_side == "left"` 时生效
  - 通过检测全宽手牌行的白色牌面总宽，减去13张手牌估算宽度，得到副露总宽
  - 除以3张牌宽度得到副露组数（0~4）

### `config/settings.yaml`
- `layout.self_hand` 新增 `meld_side: left`

**验证结果**：
- 副露=0: x=2, w=1513
- 副露=1: x=184(+182), w=1331(-182) ← 向右移动约一组副露宽度
- 副露=2: x=366(+364), w=1149(-364)

## 数据训练吧"已训练还重复出现"修复

**问题**：用户在"数据训练吧"中点击"加入训练集"后，关闭再重新打开，同样的 ROI 样本（如 1万）仍然反复出现。同时"刚刚识别的"新 ROI 没有出现在训练集队列中。

**根本原因**：`collect_training_roi_paths()` 和 `_collect_discard_roi_paths()` 中对 `processed_sources` 的过滤逻辑存在 bug：

```python
# 错误写法：最新session (sessions[0]) 永远不受已处理过滤
if source_key in processed_sources and session_name != sessions[0]:
    continue
```

等效逻辑：已处理过 AND 不是最新session → 跳过。最新 session 的数据永远不跳过，导致每次打开都重新出现。

同时因为 keyframes 分支对已处理的最新session数据不过滤，`copied` 非空，提前 `return`，导致 `recognition/` 分支中的新 ROI 无法出现。

**修复**：
- `vision/hand_region_module.py` 第201行：移除 `and session_name != sessions[0]` 特例
- `ui/main_window.py` discard 分支同样位置：移除相同特例
- 两处均改为简单的 `if source_key in processed_sources: continue`

**影响**：已标记 processed 的 keyframe source key 会被正确跳过，keyframes 分支 copied 为空时继续扫 recognition 目录，新识别的 ROI 得以出现。
