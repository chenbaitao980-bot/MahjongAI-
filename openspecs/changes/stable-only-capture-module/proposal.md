# stable-only-capture-module

## 为什么
当前主界面仍展示游戏窗口、牌面收集、区域划分、事件收集、识别运行、正式战斗、稳定版等多个入口。用户当前只需要稳定版抓包模块作为可用版本，其它模块可以废弃为不显示，但不能影响稳定版正常运行。

## 影响面
- GitNexus impact: `MainWindow`、`StableCaptureThread` 上游仅 `main.py`，风险 LOW。
- GitNexus impact: `StableBattlePanel` 上游为 `ui/main_window.py`，风险 LOW。
- 预计修改 `ui/main_window.py`，可能新增稳定版专用协调模块来隔离抓包启动、停止、消息处理。
- 不删除旧模块文件，不改变旧模块业务行为，仅从主界面入口隐藏，降低对稳定版运行的影响。

## 业务规范关系
- 命中的主 spec: `stable-reader/spec.md`
- 关系判断: Same Requirement
- 推荐动作: MODIFIED

## 改动范围
- `ui/main_window.py`
- 可能新增 `ui/stable_capture_controller.py` 或同等稳定版专用模块
- 可能调整 `ui/stable_battle_panel.py` 的轻量连接方式，但不改稳定版展示逻辑

## 验收
- [ ] 主界面只显示稳定版抓包/分析入口，不再显示游戏窗口、牌面收集、区域划分、事件收集、识别运行、正式战斗等旧入口。
- [ ] 稳定版仍可启动/停止抓包，并继续刷新 snapshot、事件、硬算/策略结果。
- [ ] 旧模块实现不被删除，避免破坏潜在依赖；稳定版运行路径与旧 UI 入口解耦。
- [ ] `python -m compileall ui stable game`
- [ ] `gitnexus detect-changes --scope all -r mahjong-learning`

## 追加需求：窗口行为优化

用户反馈稳定版界面弹出后切换到其他软件时仍然置顶，并且窗口只能上下调整大小，不能左右或斜向调整。稳定版主界面应像普通桌面窗口一样参与系统窗口层级，并支持横向、纵向和角落拖拽缩放。

### 追加影响面

- `ui/main_window.py`：移除主窗口置顶 flag，明确设置普通窗口 flag 与合理最小尺寸。
- `ui/stable_battle_panel.py`：检查稳定版面板布局约束，避免内部控件限制横向缩放。

### 追加验收

- [ ] 主窗口不再使用 `WindowStaysOnTopHint`，点击其他软件后不会继续压在最上层。
- [ ] 主窗口可以左右拖拽缩放。
- [ ] 主窗口可以从角落斜向拖拽缩放。
- [ ] 稳定版面板内容仍可正常展示，缩放时不破坏抓包操作区。

## 追加需求：模拟出牌

用户希望稳定版默认加入“模拟出牌”功能：无需进入抓包读取，也能自动模拟棋局开始并写入同结构数据；我方轮到出牌时弹出独立出牌框，用户可以和计算机模拟对战；交互、实时数据、硬算分析和推荐出牌要尽量与真实稳定版对战一致。

### 追加影响面

- `ui/stable_battle_panel.py`：增加“模拟出牌”入口信号和按钮，必要时展示模拟状态。
- `ui/main_window.py`：接入模拟模式启动、停止、用户出牌回调、电脑回合推进，并复用稳定版分析刷新路径。
- 可能新增 `stable/simulator.py` 或同等模块：维护模拟牌墙、双方手牌、弃牌、副露占位、回合、snapshot 输出。
- 可能新增 `ui/simulated_discard_dialog.py` 或同等模块：我方回合弹出可选出牌框。
- 测试新增模拟器状态流转覆盖。

### 追加验收

- [ ] 点击“模拟出牌”不会启动抓包线程，但会生成一局模拟 snapshot 并刷新稳定版实时数据。
- [ ] 模拟局开始后，界面表现与真实稳定版一致：状态、回合、手牌、弃牌、硬算分析、推荐出牌都能显示。
- [ ] 轮到我方出牌时弹出独立出牌框，用户可选择当前手牌中的一张打出。
- [ ] 用户出牌后，模拟器推进电脑回合并继续轮转到我方回合。
- [ ] 我方回合满足 14 张和可信财神时，数据分析和推荐出牌正常给出。
- [ ] 停止按钮可以结束模拟模式，不影响真实抓包模式。

## 追加需求：模拟界面舒适度

用户反馈当前默认窗口偏小、界面局促；策略建议区域上方留白过多，应把空间腾给策略建议内容；我方出牌弹框不应使用下拉框，而应像全键盘一样把所有当前手牌平铺展示，方便快速点击出牌。

### 追加影响面

- `ui/main_window.py`：调大默认窗口尺寸和合理最小尺寸。
- `ui/stable_battle_panel.py`：优化策略建议区布局，降低上方摘要框高度或调整伸缩比例，把更多空间给硬算/策略建议文本。
- `ui/simulated_discard_dialog.py`：从下拉框改为手牌按钮网格/全键盘式选择。

### 追加验收

- [ ] 主窗口默认尺寸比当前更宽更高，打开后不局促。
- [ ] 策略建议区域上方不再有大块空白，更多空间用于策略建议和硬算文本。
- [ ] 模拟出牌弹框以全手牌按钮形式展示当前手牌，而不是下拉框。
- [ ] 出牌弹框可高亮或提示推荐牌，点击任一手牌按钮即可确认出牌。

## 追加需求：模拟出牌确认

用户反馈当前模拟出牌弹框中点击手牌按钮会立即出牌，导致只是想停下来查看分析时手牌已经变化。模拟出牌应改为两步确认：点击手牌只选中，不推进局面；只有点击底部“出牌”按钮才真正出牌。关闭窗口或点击“暂停”不得改变模拟局状态。

### 追加影响面

- `ui/simulated_discard_dialog.py`：手牌按钮只负责选中；新增底部“出牌”按钮；关闭/暂停保持 reject。
- `ui/main_window.py`：保持只有 Accepted 且存在 selected_tile 时才执行 `discard_self()`。
- 测试增加弹框选择不自动 accept，以及出牌按钮才 accept 的行为。

### 追加验收

- [ ] 点击某张手牌只高亮选择，不关闭弹框，不改变手牌。
- [ ] 点击“出牌”按钮后才提交所选牌并推进模拟局。
- [ ] 关闭弹框或点击“暂停”后，模拟局 hand/discards/current_turn 不变化。
- [ ] 没有选中手牌时，“出牌”按钮不可用或不会提交。

## 追加需求：模拟吃碰杠胡事件

用户反馈模拟对局打很久也不会胡牌，也没有副露区。预检确认当前模拟器只实现发牌、摸牌、弃牌和回合轮转，`melds` 一直为空，也没有吃、碰、杠、胡事件判断。这是当前模拟模式的能力缺口。模拟对局应补齐基础吃碰杠胡事件，让交互更接近真实稳定版对战。

### 追加影响面

- `stable/simulator.py`：新增胡牌检测、吃/碰/杠合法动作检测、玩家动作应用、副露写入、手牌计数维护和事件流记录。
- `ui/main_window.py`：在模拟回合推进时弹出可选动作，处理胡/吃/碰/杠/过。
- 可能新增 `ui/simulated_action_dialog.py`：展示可选动作按钮。
- `tests/test_stable_simulator.py`：增加胡牌、副露、杠牌、过牌等状态流转覆盖。

### 追加验收

- [ ] 模拟局中可形成胡牌时，系统能识别并结束模拟局或提示胡牌。
- [ ] 对方打出可碰牌时，我方能选择碰，副露区显示碰牌。
- [ ] 对方打出可吃牌时，我方能选择吃，副露区显示吃牌。
- [ ] 我方手牌满足杠牌条件时，能选择杠，副露区显示杠牌。
- [ ] 电脑也能在基础规则下胡/碰/杠，事件流和对方副露区同步更新。
- [ ] 选择“过”时不改变手牌和副露，只继续按模拟回合推进。
