> **不建议把杭州 ECS 整体迁到香港,但可以加一台香港 VPS 作为"备用出口节点池"分流 3-5 个高风险账号。**

## 1. 网络稳定性

### 1.1 香港 → 大陆(浙江)总体网络质量

| 维度 | 实际表现 | 信心 |
|---|---|---|
| 平时延迟(浙江电信→HK) | 30-55 ms,优质 CN2 GIA 可压到 25-35 ms | 高 |
| 丢包率(平时) | CN2 GIA <0.5%,普通 BGP 1-3%,9929 中等 | 高 |
| 抖动 | CN2 GIA jitter <5ms;普通线路晚高峰 20-50ms 抖动常见 | 高 |
| GFW 干扰 | **明文 TCP 端口(7777)基本不被针对**;GFW 主要针对 VPN 协议特征(IKE/L2TP/OpenVPN/WireGuard)和 SNI/DNS 污染。游戏二进制流不属于 GFW 的主关键词扫描目标 | 中-高 |
| 节假日/敏感期 | 国庆、春节、两会期间出口带宽**普遍**收紧,普通 BGP 线路丢包飙升至 5-15%,CN2 GIA 收紧但仍可用(社区共识:"两会期间 Vultr 香港基本不可用,搬瓦工 GIA 偶有抖动但还能跑") | 高 |

### 1.2 三种线路的差异(关键)

- **CN2 GIA(中国电信下一代承载网精品线路,AS4809)**:三大运营商中**电信用户最稳**,延迟最低,丢包最少,价格最贵。GIA 与 GT 不同:**GT 高峰拥塞,GIA 永远不超卖**(理论上)。
- **9929(联通精品 AS9929)**:联通用户首选,质量接近 GIA,价格略低。
- **CMI(中国移动国际)**:移动用户首选,直连香港有专线。
- **普通 BGP / IPLC / 直连**:平时还行,**GFW 紧张期或晚高峰会被国际出口拥塞**,丢包飙升。
- **三网回程优化(三网 CN2/9929/CMI)**:理想状态,所有用户都走精品回程,搬瓦工 CN2 GIA-E 即此类。

> **对你的场景的含义**:浙江玩家用电信居多,GIA 是最优选;但游戏帧是**短包高频**(麻将出牌每秒几次),对 jitter 敏感度中等,**普通 9929 线路够用**,不必非 GIA 不可。但晚高峰/敏感期普通线路会卡顿是确定的。

### 1.3 GFW 对 HK→大陆的实际影响(给你的 7777 长连接)

- **明文自定义协议(SRS)走 7777 TCP 长连接,理论上不会被 GFW 主动 RST**(GFW 主动重置主要针对:VPN 握手特征 + 常见敏感关键词 HTTP)。
- **真实风险点**:
  - 国际出口拥塞 → 大量 packet drop → TCP 反复重传 → 应用层超时 → 游戏掉线
  - **GFW 偶发性长连接老化**:有报告(V2EX 多个帖子)指出某些线路长 TCP 连接挂 30-60 分钟后会被中间设备静默丢弃,需要 keepalive(短间隔 < 60s)续命
- **缓解**:在 noconfig 中转层启用 TCP keepalive(`SO_KEEPALIVE`,interval ≤ 30s),应用层心跳(已有 SRS 心跳更好)。

## 2. TCP 长连接稳定性

| 项目 | 结论 |
|---|---|
| 7777 端口本身 | **无敏感性**,不在 GFW 黑名单。GFW 不按端口黑名单工作,按协议特征工作 |
| 长 TCP(数小时)在香港线路 | 优质线路(CN2 GIA / 阿里云国际)上 24h 不断很常见;普通 Vultr/Linode 普通 BGP 偶有几小时一次的抖动+短暂丢包,应用层会感知为 lag spike 而非掉线 |
| 自定义 SRS 协议穿透 | **完全可以**。任何 TCP 字节流只要不带 VPN 握手特征,GFW 都不会单独识别 |
| NAT 网关空闲断流 | 国内 NAT/CGNAT 通常 5-10 分钟空闲就断,需 keepalive。已知 SRS 有自身心跳 → 不是问题 |

## 3. 大陆游戏服务端对香港 IP 的风控态度(最关键的一项)

### 3.1 行业共识(高信心,来自多年游戏运营/风控公开资料)

- **大多数国内手游/网游对海外 IP 持高度警惕**:
  - 部分游戏(腾讯、网易系)直接禁止海外 IP 登录,或要求二次验证(短信/人脸)
  - 中小厂商(包括独立麻将类)风控较弱,但仍**普遍把"地理跳变"作为风控信号**:同一账号在 1 小时内从浙江 IP 跳到香港 IP → 触发风控/封号/强制下线
  - "工作室/打金/外挂"嫌疑标签的常见命中规则:**(a) 海外 IP + (b) 固定时间段在线 + (c) 同 IP 多账号**。香港 IP 命中 (a),你之前的"同 IP 多账号"问题命中 (c) — **换 HK 反而可能命中更多规则**

### 3.2 对你具体场景(MahjongAI noconfig)的判断

| 风险维度 | 现状(杭州 ECS) | 切换香港后 |
|---|---|---|
| 同 IP 多账号(R1) | **高**(10+账号同 IP) | 同样(还是同 IP) |
| 地理跳变(R3) | **低**(浙江号配杭州 IP,符合常理) | **高**(浙江号配香港 IP,跨境) |
| "境外工作室"标签(R4) | 低 | **中-高**(海外 IP 段是已知运营标签) |
| 服务端主动黑名单海外段 | 低风险(杭州在白名单) | 中风险,**视游戏运营方而定** |

> **结论**:**单纯把出口换到香港,大概率不是降低风控,而是把"同 IP 多账号"风险换成"地理跳变 + 海外 IP"风险,综合风险可能更高,而不是更低**。

### 3.3 真正降低风控的方案(顺带说明)

1. **多机房国内 IP 池**:阿里云华东1(杭州)+ 华东2(上海)+ 华南1(深圳)+ 华北2(北京),**每个用户绑一个机房**,IP 与玩家归属地大致匹配
2. **住宅 IP 代理(如蚂蚁代理、芝麻 HTTP)**:看上去像"从家里上网",最不容易触发风控,但成本高、稳定性差
3. **香港 IP 只用于"明显是港澳台用户/外服测试"** — 你这个场景全是浙江玩家,不适合

## 6. 合规与风险提示

### 6.1 法规

- **大陆个人购买香港 VPS 用于个人用途(非建立中文网站对内服务)**:**不触碰任何明确法规**。香港 VPS 不在大陆境内,**无需 ICP 备案**(只有大陆境内服务器对内提供网站服务才需备案)。
- **实名要求**:
  - 国内厂商(阿里云/腾讯云/华为云)**国内账号买香港轻量也要实名**(身份证),这是国内云服务统一要求,与 IP 所在地无关。
  - 海外厂商(BWG/RackNerd/Vultr 等):**只需邮箱+支付方式**,不要身份证。
- **使用合规**:
  - ⚠ 不得用香港 VPS 搭建对大陆境内的科学上网服务(VPN/代理给他人用),这会触碰《反电信网络诈骗法》《刑法 285 条》;**自用浏览不犯法,商业经营才犯法**(2024 后多个判例确认)
  - 你的场景(noconfig 中转游戏数据,自用)**不属于科学上网经营**,法律风险极低。但要注意:**多用户 + 收费 = 经营**,即使技术上不是 VPN,**"提供给多用户的境外网络服务"在 2024 年起被部分地方司法解释纳入"非法经营"** — 这点须谨慎,如果你给 10+ 用户提供付费 noconfig 服务,**法律边界开始模糊**

### 6.2 风险三连

1. **R1(同 IP 多账号)**:换 HK 不解决,需要 IP 池
2. **R3(地理跳变)**:换 HK **新增此风险**,大陆游戏运营方对"浙江实名号 + 海外 IP"有规则化的风控扣分
3. **R4(运营经营性认定)**:多用户付费的"境外中转"服务,2024 年后司法风险上行;**技术上是中转,法律上可能被定性为"非法经营国际电信业务"**
