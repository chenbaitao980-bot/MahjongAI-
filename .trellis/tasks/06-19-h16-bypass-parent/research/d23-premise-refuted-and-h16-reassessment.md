# D23 前提证伪 + H16 父任务态势重估 (2026-06-20 凌晨自主分析)

> 用户已睡，授权自主推进。本文件记录今晚离线分析（不需用户配合）得出的关键结论。

## TL;DR

**D23 的核心前提是错的。** D23 假设「0x2BC0 sub_cmd=0x022B = 局末摊牌帧，含双方完整 14 张手牌，>80% 命中」。但实证（2026-06-16 真机 + 今晚复核两个回放样本）证明：

**0x022B = `deal_1v1`，只含玩家自己的 13 张起手牌，没有对手数据。** 它是 1v1 好友房的发牌帧（等价于金币局的 0x0003 deal），不是局末摊牌。

protocol.py:62 把它误标为 `"round_result"` —— 这个错误标签正是 D23 立项的依据。但 memory [[stable-022b-1v1-deal]]（2026-06-16 实锤）早已纠正为 `deal_1v1`，D23 立项时没参考到这条。

## 证据链

### 1. 回放样本里根本没有 0x022B
两个归档样本（26K + 33K，spectator 抓的完整一局）：
- 原始字节扫描 `2b 02`（0x022B LE）：**0 次出现**
- 事件流枚举：8× deal(0x0003) + 1× win(0x0220) + 大量 draw/discard/hand_update/stat
- **没有任何 0x022B 帧**，也没有第二个真手牌块

样本是金币局（走 0x0003 deal），0x022B 只在好友房 1v1 出现。

### 2. 0x022B 的真实结构（2026-06-16 真机抓取，live-frames-2026-06-16.md）
```
2b02 6700 10000000 00 0d <13B block1> 00 0d <13B> 00 0d <13B> 00 0d <12B> 38 ...
```
- block1 13 个 instance id 全互不重复 = **玩家自己的真手牌**
- block2/3/4 id 有重复 = 被误切的开局数据，**不是对手手牌**
- 服务器**只下发本人手牌**，无 player 字段

### 3. win 帧 (0x0220) 也不是摊牌
23B 结算摘要（`08000000...41010300...`），无手牌 tile 数据。

### 4. spectator 样本全程 view-filtered
- deal @306: 自己 13 张真牌 + 对手位 `7×0x3c` 占位
- hand_update player=1（对手）: 要么 count=1 占位、要么 `72 72 72...`(0x72 越界=隐藏)
- **对手真手牌在任何帧都不出现** —— 服务端 H16 view filter 是全局的

## 对 H16 父任务的影响

三个子任务重估：

| 子任务 | 原假设 | 实证结论 |
|--------|--------|----------|
| **D23** (0x022B 摊牌) | >80% 拿双方手牌 | **证伪**：0x022B 只含自己手牌，无摊牌帧存在 |
| **N6** (隐藏协议扫) | 15-30% 找到 dump 协议 | 进行中：服务端对 unknown mt **静默**，命中面窄 |
| **N5** (侧信道听牌) | 弱信号降级 | 仍可做（离线分析 hand_update 元信息）|

**核心障碍 H16 比预想更硬**：服务端在协议层**从不下发对手手牌真值**——不是某个帧能绕过的过滤，而是整个 spectator/replay 数据管道在服务端序列化时就把对手 hand 替换成 0x3c。

**最强证据**：26K 样本是金币局**打到分胜负**的完整一局（含 win 帧 0x0220）。即便局已结束，回放数据里对手手牌**仍是 0x3c 占位**，胜负结算走纯数值 win 帧（23B，不亮牌）。说明这个游戏**没有「局末亮对手牌」这种 UI 帧**。

**一个残留 caveat**：2026-06-16 的好友房 1v1 真机抓取只到中盘（discard 阶段），未抓到 1v1 打到胡牌/流局的局末时刻。理论上 1v1 局末**可能**有一个不同于 0x022B 的摊牌帧（毕竟 1v1 UI 会亮对手牌让人服气）。但金币局的全局证据强烈暗示该游戏服务端不亮对手牌。要彻底排除需打一局 1v1 到胡牌并抓帧（需用户配合，已记入 NEXT-STEPS）。
