# stable-reader-hard-analysis-panel

## 为什么

稳定版右侧区域此前展示“策略建议”和“候选分析”，其中会混入 AI/MC/危险度等软判断。用户要求暂时去掉右侧策略建议和分析，替换为计算机可以根据抓包手牌硬算出来的内容，并且必须满足：

1. 右侧计算数据与 UI 左侧展示数据完全一致，不能另取一套状态。
2. 计算逻辑单独抽模块，避免散落在 UI。
3. 数据不足时明确指出，不乱给建议。
4. 建议理由只使用可验证规则：进听、有效进张最多、不打财神、不退听。

## 预检证据

- `ui/stable_battle_panel.py` 右侧原来复用 `AnalysisPanel`，展示 AI/候选分析表。
- `stable/tracker.py::snapshot()` 已输出左侧展示所需的同一份手牌、副露、弃牌、财神、当前回合、可信度字段。
- `game/shanten.py::calc_shanten()` 已提供财神 wildcard 向听计算，影响面较高，不应修改本体。
- `game/ukeire.py::calc_ukeire()` 已提供打出后有效进张计算，可复用。
- 当前 active change `stable-reader-optional-action-round2-fixes` 已完成可选动作/等待态/LLM 兜底任务，但不包含本次硬算面板需求；本 change 是后续独立需求。

## 影响面

| 文件 | 改动 |
| --- | --- |
| `game/stable_hard_analysis.py` | 新增稳定版硬算模块，基于 snapshot 计算状态、财神、向听、听牌、进张、建议、风险、可信度 |
| `ui/stable_battle_panel.py` | 右侧由 AI 候选分析表改为硬算文本面板，AI 返回不再覆盖硬算结果 |
| `tests/test_stable_hard_analysis.py` | 增加数据不足与 14 张手牌硬算建议回归测试 |

## 验收

- [x] 右侧显示当前状态、财神、当前向听、是否听牌、听牌列表、最佳进听打法、有效进张、当前建议、建议原因、强提醒、财神风险、数据可信度。
- [x] 右侧计算仅使用 `snapshot()` 中同一份手牌/副露/弃牌/财神/回合数据。
- [x] 财神未解析、手牌不可信、未知牌值或当前事件不可信时，显示数据不足，不给硬建议。
- [x] 向听计算复用 `calc_shanten()`，不修改共享向听算法。
- [x] 听牌枚举通过 `getTingTiles()`/`get_ting_tiles()` 完成。
- [x] 右侧不再展示旧 `AnalysisPanel` 候选分析表。

## 追加需求：对方预测展示

用户要求确认 UI 是否展示“对方手牌可能性预测 / 对方进度预测”。当前稳定版面板只展示对方已知手牌计数、弃牌、副露和等待态，右侧硬算只覆盖我方当前向听、听牌、进张、推荐与候选重排，没有明确展示对方隐藏手牌可能性或对方进度预测。

### 追加影响面

| 文件 | 改动 |
| --- | --- |
| `game/stable_hard_analysis.py` | 在稳定版硬算结果中增加对方预测字段，基于 snapshot 的对方弃牌、副露、可见牌、剩余牌做保守估计 |
| `ui/stable_battle_panel.py` | 在右侧硬算文本中展示对方手牌可能性预测和对方进度预测 |
| `tests/test_stable_hard_analysis.py` | 增加对方预测字段的回归测试 |

### 追加验收

- [ ] 右侧稳定版硬算面板展示“对方手牌可能性预测”。
- [ ] 右侧稳定版硬算面板展示“对方进度预测”。
- [ ] 预测文案明确是估计，不展示对方隐藏手牌为确定值。
- [ ] 对方预测只使用当前 `snapshot()` 中可见信息：对方弃牌、副露、我方手牌、已见牌、剩余牌和当前回合。

## 追加需求：策略建议区域和数据颜色优化

用户反馈右侧“策略建议”区域当前位置偏低，底部硬算数据容易被截断；同时希望下方策略/数据内容按照轻重缓急和种类用不同颜色展示，便于扫读。

### 追加影响面

| 文件 | 改动 |
| --- | --- |
| `ui/stable_battle_panel.py` | 调整稳定版面板右侧策略区域布局权重/高度，让策略建议向上扩张并显示完整底部数据；把硬算明细从纯文本改为可着色富文本或等价展示 |

### 追加验收

- [ ] “策略建议”区域顶部向上扩张，占用右侧可用高度，底部硬算数据不再被窗口底部截断。
- [ ] 策略明细按轻重缓急/数据种类使用不同颜色展示，例如关键提醒/风险为红色或橙色，建议/推荐为绿色或蓝色，状态/预测/可信度为不同辅助色。
- [ ] 左侧实时数据和事件流仍保持可用，不引入新的业务分析字段或策略行为。
